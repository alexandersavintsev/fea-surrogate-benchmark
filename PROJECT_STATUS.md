# PROJECT STATUS — суррогаты для МКЭ (Junior ML Contest, ИТМО)

Живой ориентир по проекту. Обновляется по ходу.

## Находка (ядро проекта)

Выбор регрессионного суррогата определяется гладкостью отклика:
на гладких упругих задачах лучшая - GPR; на негладких/пороговых (пластичность,
разрывы, резонанс) выигрывают ансамбли деревьев (XGBoost / LightGBM / CatBoost).

## Задачи (4), центр - пара упругая/пластическая консоль

| Задача | Режим | Ожидание | Роль |
|---|---|---|---|
| Кирш (пластина с отверстием) | гладкий упругий | GPR | опорная гладкая задача |
| Консоль упругая (80/120/240) | гладкий | GPR (гипотеза о деревьях НЕ подтвердилась) | «до» в паре |
| Консоль пластическая (порог текучести) | негладкий | деревья | «после» в паре, GPR-fail кейс |
| Синтетика (гладкая/разрывная/ВЧ/резонанс) | оба | контроль механизма | чистое доказательство границы |

Та же геометрия консоли, меняется только режим физики - победитель переворачивается.

## Код (готово)

- `src/synthetic.py` - синтетические датасеты (проверено).
- `src/models.py` - 8 моделей: Ridge, GPR, MLP, RandomForest, sklearn-GB, XGBoost,
  LightGBM, CatBoost (последние три - опциональные импорты).
- `src/benchmark.py` - k-fold CV, метрики R2/MAE/RMSE/MAPE, флаги --csv/--target/--drop.
- `src/run_all.py` - ОДНА команда: прогон всех задач -> `results/crossover_all.csv`
  + печать кроссовера (GPR против лучшего дерева). Привязка признаков/целей выверена.
- `src/plot_crossover.py` - итоговый график `results/crossover.png`.
- `fea/optimize.py` (демо) и `fea/optimization/optimize_surrogate.py` (реальная, Кирш).
- `fea/generate_plastic_doe.py` - генератор пластической выборки с авто-проверкой.
- `apdl/plastic_cantilever.mac` - параметрический макрос упругопластической консоли.
- Обвязка: Dockerfile, `.github/workflows/ci.yml`, `app/streamlit_app.py`, tests, README.

## Что осталось

1. Прогнать `apdl/plastic_cantilever.mac` по `data/plastic_doe_inputs.csv` в ANSYS,
   собрать результаты в `data/plastic_results.csv` (колонки: len,height,thk,force,
   sy,epl_max,uy_tip,sig_max,area). Пилот на ~20 точках, потом все.
2. `pip install -r requirements.txt`, затем `python -m src.run_all` и
   `python -m src.plot_crossover` - получить сводную таблицу и график кроссовера.
3. Optuna-тюнинг, MLflow-трекинг.
4. Оптимизация: причесать Кирш, распространить на пластику, Парето, проверка в ANSYS.
5. git push на GitHub (init делать на Windows), Docker/CI зелёные.
6. Упаковка: описание проекта (PDF ≤5 стр.), презентация (шаблон ИТМО), CV, мотивационное.

## РЕЗУЛЬТАТЫ (подтверждено, R² по k-fold CV)

Находка: выбор суррогата определяется гладкостью отклика. На гладких упругих задачах
и даже при пластичности лучшая - GPR (на малых выборках 30-100 точек разносит бустинг).
Но на настоящей негладкости - резонансе - GPR теряет первое место, её обгоняет
градиентный бустинг, и это устойчиво к тюнингу Optuna.

Гладкие (побеждает GPR):
- kirsch_100: GPR 1.00 | лучший бустинг ~0.96
- kirsch_30 (два сида): GPR 0.99 | деревья 0.06-0.78
- cantilever 80/120/240: GPR 0.99 / 0.76 / 0.90 | лучший бустинг 0.80 / 0.76 / 0.81

Негладкие:
- plastic (мягкий излом): GPR 0.99 | бустинг <= 0.85 (GPR ещё держится)
- resonance: CatBoost 0.61 (tuned 0.61), LightGBM 0.49  >  GPR 0.45  -> КРОССОВЕР
- синтетика: threshold -> GPR; step / resonance -> деревья/MLP

Ключевой вывод для питча: дефолт для гладких МКЭ-суррогатов - GPR; для резких/резонансных
откликов - градиентный бустинг. Показано на реальной КЭ-задаче, не только на синтетике.

Оптимизация: `fea/optimization/optimize_surrogate.py` - минимизация напряжения/площади
на GPR-суррогате Кирша + проверка оптимума в ANSYS.
