# Surrogate models for FEA: when GPR wins and when gradient boosting wins

Бенчмарк регрессионных суррогатов конечно-элементного анализа и инструмент построения
суррогата с последующей инженерной оптимизацией.

**Находка:** выбор модели-суррогата определяется гладкостью отклика. На гладких упругих
задачах и малых выборках лучшая - регрессия гауссовских процессов (GPR): она обгоняет даже
современный градиентный бустинг. Но на резком (резонансном) отклике GPR теряет первое место -
её обходит бустинг (CatBoost R² = 0,61 против GPR 0,45). Показано на реальных данных ANSYS.

![Кроссовер: GPR против градиентного бустинга по задачам разной гладкости](docs/figs/fig_crossover.png)

## Структура

```
src/            ядро: данные, модели, бенчмарк, оптимизация
  synthetic.py  синтетические датасеты с контролируемой гладкостью
  models.py     зоопарк моделей (8 шт.)
  benchmark.py  k-fold кросс-валидация + метрики, сохраняет результаты
  optimize.py   оптимизация конструкции на обученном суррогате
apdl/           параметрические макросы ANSYS APDL (упругопластическая консоль)
app/            Streamlit-демо
configs/        конфигурация эксперимента
tests/          smoke-тесты
```

## Быстрый старт

```bash
pip install -r requirements.txt
python -m src.benchmark            # синтетический бенчмарк -> results/benchmark.csv
python -m src.optimize             # демо оптимизации на суррогате
streamlit run app/streamlit_app.py # интерактивное демо
pytest -q                          # тесты
```

Docker:

```bash
docker build -t surrogate .
docker run --rm -v $PWD/results:/app/results surrogate
```

## Данные ANSYS

Макрос `apdl/plastic_cantilever.mac` считает упругопластическую консоль и пишет
строку результата в `plastic_result.txt`. Параметрический прогон по выборке LHS
делается тем же батч-скриптом, что и в основной работе (подстановка параметров +
запуск в batch-режиме), результаты собираются в CSV и подаются в `src/benchmark.py`.

## Метрики

R², MAE, RMSE, MAPE; сравнение по k-fold CV (несколько сидов), статистика различий.
