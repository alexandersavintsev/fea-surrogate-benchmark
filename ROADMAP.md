# Дорожная карта проекта на Junior ML Contest (ИТМО, 3-я волна)

**Личный дедлайн: 14 июля (всё на руках).** Окно 3-й волны: подача 3-20 июля,
защита 27-29 июля. Питч: 5 мин + 7 мин вопросов.

## Идея (перепаковка проекта)

Не «сравнил модели», а исследование с находкой:
**«Когда суррогат для МКЭ лучше строить деревьями, а когда гауссовским процессом -
граница проходит по гладкости отклика»**, плюс рабочий инструмент построения
суррогата и инженерная оптимизация на нём.

Три кейса:
1. **Гладкая упругая задача** (Кирш + линейная консоль) - выигрывает GPR. *Есть данные.*
2. **Негладкая нелинейная задача** - упругопластическая консоль с порогом текучести.
   Отклик (макс. пластическая деформация) равен нулю до текучести и резко растёт после.
   Тут выигрывают деревья (XGBoost / LightGBM / CatBoost). *Новое, ядро работы.*
3. **Синтетический бенчмарк по гладкости** - гладкая / разрывная / высокочастотная /
   резонансный пик. Чисто и воспроизводимо показывает границу. *Готово (`src/`).*

Поверх лучшего суррогата - оптимизация конструкции с проверкой полным КЭ-расчётом.

## Соответствие критериям конкурса

- **Data Science**: 8 моделей, k-fold CV, Optuna-тюнинг, несколько сидов, метрики
  R²/MAE/RMSE/MAPE, находка-кроссовер.
- **Разработка и инженерия**: GitHub, чистая структура, config-driven пайплайн,
  Docker, GitHub Actions (линт+тесты), MLflow, unit-тесты.
- **Применение ИИ**: задокументировать использование AI-ассистентов/агентов.
- **Продукт**: ЦА - инженеры-расчётчики; проблема - МКЭ дорог для многовариантных
  задач; MVP - Streamlit-инструмент; импакт - ~1000x ускорение; фидбэк от инженеров.
- **Мотивация**: связать механику + ML с AI Talent Hub.

## Ключевой нарратив для питча

«Суррогат ускоряет МКЭ на порядки, но выбор модели зависит от гладкости отклика:
на гладких упругих задачах лучшая - GPR, на негладких/пороговых (пластичность) -
градиентный бустинг. Я показал эту границу на синтетике и на реальной КЭ-задаче и
применил лучший суррогат для оптимизации конструкции с проверкой полным расчётом.»
